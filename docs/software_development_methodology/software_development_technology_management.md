### アーキテクトスキルとは

システムの設計から運用保守までをリードすることである。

<br>

### より具体的に

より具体的には、アーキテクトスキルは以下に分けられる。

図とは若干用語の表記を変えており、実装も加えている。

1. 要件定義 (窓口的に課題をヒアリングする)
   - ドメイン
   - ユースケース
2. デザイン
   - UI
3. システム設計
   - 技術選定
   - 必要な設計ドキュメントの整備
   - その他開発ルールの整備
4. 実装
   - 機能追加
   - 変更
   - 修正
5. テスト
   - テストコード
   - テスト方法の品質担保
6. リリース
   - リリース方法の品質担保
7. 運用保守
   - 監視体制の整備
   - リリース規約
   - アップグレード規約
