# 運用設計＠AI用ドキュメント

## セキュリティ運用

### マルウェア対策

| 運用設計項目                                               | 中央集中型の例                                                                                         | 分散型の例                                                           |
| ---------------------------------------------------------- | ------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------- |
| コンテナイメージのOS・言語・ライブラリ・依存パッケージ更新 | すべてのマイクロサービスでOSや言語、依存ライブラリのバージョンを統一し、月次または四半期ごとに更新する | 各マイクロサービスが自律的にライブラリを更新し、脆弱性を都度修正する |
| 脆弱性スキャン運用                                         | すべてのマイクロサービスでCI/CDにスキャンを統合し、結果を中央で集約して分析する                        | 各マイクロサービスが自律的にスキャンを実施し、修正対応を行う         |
| マルウェア検知運用                                         | すべてのマイクロサービスの通信とファイルをセキュリティ基盤で自動スキャンし、結果を中央で管理する       | 各マイクロサービスが自律的にログを分析し、異常を検知する             |
| マルウェア定義更新                                         | すべてのマイクロサービスで使用する検知ルールやシグネチャを統一し、セキュリティチームが一括更新する     | 各マイクロサービスが自律的に独自定義を必要に応じて更新する           |

<br>

### アプリケーションデータの暗号化

| 運用設計項目               | 中央集中型の例                                                                            | 分散型の例                                                           |
| -------------------------- | ----------------------------------------------------------------------------------------- | -------------------------------------------------------------------- |
| 証明書ローテーション       | すべてのマイクロサービスで証明書の更新周期を統一し、30日から90日ごとに自動更新する        | 各マイクロサービスが自律的に有効期限前に手動で更新する               |
| 鍵・トークンローテーション | すべてのマイクロサービスでVaultやSecret Managerを利用し、自動ローテーションを統一運用する | 各マイクロサービスが自律的にSecretを管理し、手動で更新する           |
| 通信経路暗号化             | すべてのマイクロサービス間通信をTLSで統一し、IstioやIngress Gatewayで中央管理する         | 各マイクロサービスが自律的にTLS設定を実施し、証明書を管理する        |
| 一時ファイル暗号化         | すべてのマイクロサービスで暗号化ルールを統一し、共通ボリュームを保護する                  | 各マイクロサービスが自律的にアプリケーション側で暗号化実装を管理する |
| 暗号化ログ監査             | すべてのマイクロサービスで暗号化通信ログを中央に収集し、定期的に検証を実施する            | 各マイクロサービスが自律的にログを監査し、改善を進める               |

<br>

### 保管データの暗号化

| 運用設計項目             | 中央集中型の例                                                                                 | 分散型の例                                                             |
| ------------------------ | ---------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------- |
| ストレージ暗号化設定運用 | すべてのマイクロサービスでストレージ暗号化設定を統一し、KMSやCloud KMSを中央で管理する         | 各マイクロサービスが自律的にEBSやS3バケットの暗号化設定を管理する      |
| 鍵管理運用               | すべてのマイクロサービスでKMSキーのライフサイクルを統一し、セキュリティ部門が中央で管理する    | 各マイクロサービスが自律的にサービス専用のKMSキーを管理する            |
| 鍵ローテーション         | すべてのマイクロサービスで鍵ローテーション周期を統一し、KMSやHSMで自動ローテーションを実施する | 各マイクロサービスが自律的にスケジュールを定めて鍵をローテーションする |
| 暗号化状態の監査         | すべてのマイクロサービスでストレージの暗号化状態を定期的にチェックし、レポートを中央で集約する | 各マイクロサービスが自律的に暗号化設定を点検する                       |
| バックアップデータ暗号化 | すべてのマイクロサービスでバックアップデータの暗号化方式を統一し、暗号化を自動適用する         | 各マイクロサービスが自律的にバックアップ処理に暗号化を設定する         |

<br>

### 認証／認可

| 運用設計項目         | 中央集中型の例                                                                               | 分散型の例                                                         |
| -------------------- | -------------------------------------------------------------------------------------------- | ------------------------------------------------------------------ |
| IDプロバイダ管理     | すべてのマイクロサービスで統一されたIDプロバイダを利用し、認証を一元管理する                 | 各マイクロサービスが自律的にサービス専用のID管理を行う             |
| 認証鍵ローテーション | すべてのマイクロサービスでAPIキーやSSO鍵のローテーション周期を統一し、自動実行する           | 各マイクロサービスが自律的に鍵を更新する                           |
| アクセス権限レビュー | すべてのマイクロサービスでアクセス権限レビューの周期を統一し、セキュリティ部門が定期点検する | 各マイクロサービスが自律的にメンバー権限を見直す                   |
| 認可ポリシー管理     | すべてのマイクロサービスでIAMやRBACの設定変更を中央承認制で統一する                          | 各マイクロサービスが自律的にNamespace単位でロールを管理する        |
| 認証ログ監査         | すべてのマイクロサービスで認証ログを中央監査基盤に集約し、異常を検知する                     | 各マイクロサービスが自律的にログを監視し、不審なアクセスを検出する |

## CI/CD運用

| 運用設計項目           | 中央集中型の例                                                                       | 分散型の例                                                   |
| ---------------------- | ------------------------------------------------------------------------------------ | ------------------------------------------------------------ |
| パイプライン保守周期   | すべてのマイクロサービスでパイプライン構成を統一し、半期ごとにテンプレートを更新する | 各マイクロサービスが自律的にパイプラインを更新する           |
| リリースウィンドウ設定 | すべてのマイクロサービスで本番反映時間帯を統一し、中央でスケジュール管理する         | 各マイクロサービスが自律的に任意の時間にデプロイを実施する   |
| CI/CDトークン更新      | すべてのマイクロサービスで認証情報の更新周期を統一し、中央で一元管理する             | 各マイクロサービスが自律的にリポジトリ内の認証情報を更新する |
| リリース承認フロー     | すべてのマイクロサービスで承認フローを統一し、中央でレビューを実施する               | 各マイクロサービスが自律的にセルフ承認でデプロイを進める     |
| パイプライン障害対応   | すべてのマイクロサービスでCIエラー検知ルールを統一し、中央が修正対応を行う           | 各マイクロサービスが自律的に失敗を検知し、再実行する         |

<br>

## 監視運用

| 運用設計項目               | 中央集中型の例                                                                                 | 分散型の例                                               |
| -------------------------- | ---------------------------------------------------------------------------------------------- | -------------------------------------------------------- |
| アラート閾値見直しサイクル | すべてのマイクロサービスでアラート閾値の見直し周期を統一し、四半期ごとに全体レビューを実施する | 各マイクロサービスが自律的に閾値を調整する               |
| SLOレポート更新頻度        | すべてのマイクロサービスでSLOレポートの更新頻度を統一し、月次で自動生成・共有する              | 各マイクロサービスが自律的にレポートを作成し、共有する   |
| ダッシュボード保守運用     | すべてのマイクロサービスで監視ダッシュボード構成を統一し、中央が定期更新する                   | 各マイクロサービスが自律的にウィジェットを編集・削除する |
| アラート対応運用           | すべてのマイクロサービスでアラート対応手順を統一し、中央当番が一次対応を行う                   | 各マイクロサービスが自律的に一次対応と復旧を行う         |
| 監視ルール改善活動         | すべてのマイクロサービスで監視ルールの改善サイクルを統一し、中央で定期レビューを実施する       | 各マイクロサービスが自律的に改善サイクルを運用する       |

## インシデント運用

| 運用設計項目             | 中央集中型の例                                                                                               | 分散型の例                                                         |
| ------------------------ | ------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------ |
| オンコールシフト更新頻度 | すべてのマイクロサービスでオンコール体制の更新周期を統一し、月次または四半期ごとに中央でローテーションを行う | 各マイクロサービスが自律的に当番表を管理し、柔軟に交代する         |
| 初動対応手順運用         | すべてのマイクロサービスで初動対応手順を統一し、共通Runbookを定期更新する                                    | 各マイクロサービスが自律的にRunbookを運用し、必要に応じて更新する  |
| エスカレーションルール   | すべてのマイクロサービスでエスカレーションルールを統一し、中央が連絡経路と優先度を管理する                   | 各マイクロサービスが自律的に報告経路を判断し、必要に応じて連携する |
| ポストモーテム運用       | すべてのマイクロサービスでポストモーテム手順を統一し、中央が全体レビューを実施する                           | 各マイクロサービスが自律的に振り返りを行い、改善策をまとめる       |
| 障害演習・訓練周期       | すべてのマイクロサービスで演習スケジュールを統一し、年2回の全社的訓練を実施する                              | 各マイクロサービスが自律的に小規模演習を企画・実施する             |
