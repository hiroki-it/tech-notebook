# テストコード仕様のドキュメント

このドキュメントは、テストコード仕様を考えるためのドキュメントであり、生成AIに読み込ませることを想定している。

1. テスト対象の実装を読み込む。
2. テストコードを実装し、次の表を作成してください。
   1. パブリックな処理をテストすることで、プライベートな処理も間接的にテストできるため、プライベートな処理内部のテストコードは不要です
   2. テストケース名は `should return <返却値> when <入力値>` とし、テスト対象のコードは変更しないこと。

| テストスイート | 種別   | テストケース名                            | テスト内容の概要                             |
| -------------- | ------ | ----------------------------------------- | -------------------------------------------- |
| foo            | 正常系 | should return user when userId is valid   | 正しいUserIdだった場合にUserを返却すること   |
| bar            | 異常系 | should throw error when userId is invalid | 不正なUserIdだった場合にUserをスローすること |
