    設定ファイル＠uWSGI¶

    はじめに¶

    本サイトにつきまして、以下をご認識のほど宜しくお願いいたします。

    • https://hiroki-it.github.io/tech-notebook/


    01. セットアップ¶

    インストール¶

    ▼ pipリポジトリから¶

    $ pip3 install uwsgi


    02. 設定ファイルの種類¶

    uwsgi.iniファイル¶

    ▼ uwsgi.iniファイルとは¶

    uWSGIの起動時の値を設定する。

    .json形式やXML形式でも問題ない。

    • https://uwsgijapanese.readthedocs.io/ja/latest/Options.html
    • https://qiita.com/11ohina017/items/da2ae5b039257752e558

    04. uwsgiセクション¶

    uwsgiセクションとは¶

    uWSGIのuwsgiプロセスのプールを設定する。


    callable¶

    アプリケーションのインスタンスの変数名を設定する。

    デフォルト値は、applicationである。

    [uwsgi]
callable = app
    • https://laplace-daemon.com/nginx-uwsgi-flask/
    • https://uwsgijapanese.readthedocs.io/ja/latest/Options.html#callable


    chdir¶

    作業ディレクトリから移動する。

    [uwsgi]
chdir=/var/www/foo
    • https://uwsgijapanese.readthedocs.io/ja/latest/Options.html#chdir


    chmod-socket¶

    Unixドメインソケットファイルの権限を設定する。

    [uwsgi]
chmod-socket = 666
    • https://uwsgijapanese.readthedocs.io/ja/latest/Options.html#chmod-socket


    die-on-term¶

    [uwsgi]
die-on-term = true
    • https://uwsgijapanese.readthedocs.io/ja/latest/Options.html#die-on-term


    http¶

    HTTPリクエストを使用する場合、受信するインバウンド通信のIPアドレスと、インバウンド通信を待ち受けるポート番号を設定する。

    Pythonアプリケーションをフレームワークで開発している場合は、フレームワークのデフォルトのポート番号を指定する。

    • https://uwsgijapanese.readthedocs.io/ja/latest/Options.html#http
    [uwsgi]
http = 0.0.0.0:5000


    logto¶

    ログの出力先を設定する。

    [uwsgi]
logto = /dev/stdout
    • https://uwsgijapanese.readthedocs.io/ja/latest/Options.html#logto


    master¶

    マスターモードで起動するか否かを設定する。

    [uwsgi]
master = true
    • https://uwsgijapanese.readthedocs.io/ja/latest/Options.html#master


    module、wsgi¶

    Pythonでアプリケーションファクトリーパターンを採用している場合、エントリーポイントのディレクトリ名とファクトリーメソッド名を設定する。

    [uwsgi]
module = src:create_app()
    • https://uwsgijapanese.readthedocs.io/ja/latest/Options.html#module-wsgi


    processes¶

    [uwsgi]
processes = 1
    • https://uwsgijapanese.readthedocs.io/ja/latest/Options.html#processes


    py-autoreload¶

    [uwsgi]
py-autoreload = 1
    • https://uwsgijapanese.readthedocs.io/ja/latest/Options.html#py-autoreload


    python-path¶

    アプリケーションのあるディレクトリを設定する。

    [uwsgi]
python-path = /var/www/foo
    • https://uwsgijapanese.readthedocs.io/ja/latest/Options.html#python-path


    socket¶

    Unixドメインソケットを使用する場合、ソケットファイルの作成場所と、インバウンド通信を待ち受けるポート番号を設定する。

    Pythonアプリケーションをフレームワークで開発している場合は、フレームワークのデフォルトのポート番号を指定する。

    [uwsgi]
socket = /etc/uwsgi/uwsgi.sock:5000
    • https://uwsgijapanese.readthedocs.io/ja/latest/Options.html#socket
    • https://qiita.com/koyoru1214/items/57461b920dfc11f67683


    vacuum¶

    uwsgiプロセス終了時にソケットファイルを削除するか否かを設定する。

    [uwsgi]
vacuum = true
    • https://uwsgijapanese.readthedocs.io/ja/latest/Options.html#vacuum


    wsgi-file¶

    エントリーポイントとするファイルを設定する。

    [uwsgi]
wsgi-file = main.py
    • https://uwsgijapanese.readthedocs.io/ja/latest/Options.html#wsgi-file
    • https://django.kurodigi.com/uwsgi-basic/
