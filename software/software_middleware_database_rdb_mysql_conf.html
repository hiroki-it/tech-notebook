    設定ファイル＠MySQL¶

    はじめに¶

    本サイトにつきまして、以下をご認識のほど宜しくお願いいたします。

    • https://hiroki-it.github.io/tech-notebook/


    01. セットアップ¶

    インストール¶

    ▼ dnfリポジトリから¶

    mysqlコマンドのみをインストールしたい場合はこちら。

    $ dnf install -y mysql

    mysqlコマンド、MySQLのRDBMS、をインストールしたい場合はこちら。

    $ dnf install -y mysql-server

    ▼ yumリポジトリから¶

    mysqlコマンドのみをインストールしたい場合はこちら。

    $ yum install -y mysql

    mysqlコマンド、MySQLのRDBMS、をインストールしたい場合はこちら。

    $ yum install -y mysql mysql-server
    • https://qiita.com/gologo13/items/1bdba6085ec79153bf1a


    動作確認¶

    ▼ ステータス確認¶

    MySQLのプロセスが稼働していることを確認する。

    $ systemctl status mysql.service

    ▼ コネクション確認¶

    DBに接続する。

    pオプションの値にはスペースが不要であることに注意する。

    $ mysql -u <ユーザー名> -p<パスワード> -h <DBホスト名> <DB名>


    パラメーター¶

    ▼ パラメーターの表示¶

    DBに登録されているグローバルパラメーターとセッションパラメーターを取得する。

    -- セッション/グローバルパラメーターを表示
SHOW SESSION VARIABLES;
SHOW GLOBAL VARIABLES;

-- OSとDBのタイムゾーンに関するパラメーターを表示
SHOW SESSION VARIABLES LIKE "%time_zone";
SHOW GLOBAL VARIABLES LIKE "%time_zone";

    ▼ パラメーターの設定¶

    -- グローバルパラメーターの場合
SET GLOBAL time_zone = "Asia/Tokyo";

-- セッションパラメーターの場合
SET time_zone = "Asia/Tokyo";


    02. 設定ファイルの種類 (※ Dockerの場合)¶

    /etc/mysql/my.cnfファイル¶

    ▼ /etc/mysql/my.cnfファイルとは¶

    MySQLの起動時の値を設定する。

    my.cnfファイルは、/etc/mysqlディレクトリ配下に配置されている。

    ただ実際には、/etc/mysql/conf.d/ディレクトリと/etc/mysql/mysql.conf.d/ディレクトリ配下にある*.cnfファイルを読み込むようになっている。

    03. mysqldセクション¶

    mysqldセクション¶

    mysqlサーバーのmysqldプロセスのプールを設定する。


    character-set-server¶

    ▼ とは¶

    DBの作成時に適用する文字コードを設定する。

    [mysql]
character-set-server = utf8mb4


    collation_server¶

    ▼ とは¶

    照合順序を設定する。

    [mysqld]
collation_server = utf8mb4_general_ci


    default-time-zone¶

    ▼ default-time-zoneとは¶

    デフォルトのタイムゾーンを設定する。

    [mysqld]
default-time-zone = SYSTEM


    datadir¶

    ▼ datadirとは¶

    DBの定義ファイルを配置するディレクトリを設定する。

    [mysqld]
datadir = /var/lib/mysql

    ▼ datadirディレクトリ¶

    DBの定義ファイルを管理する。

    基本的には/var/lib/mysqlディレクトリを使用するように設定されている。

    DB固有の情報は、DB名のディレクトリで管理されている。

    dockerエリアのマウントポイントとして指定される。

    • https://dev.mysql.com/doc/refman/8.0/ja/data-directory.html

    log-error¶

    ▼ log-errorとは¶

    エラーログの出力先を設定する。

    [mysqld]
log-error = mysql-error.log


    log_timestamps¶

    ▼ log_timestampsとは¶

    ログのタイムゾーンを設定する。

    [mysqld]
log_timestamps = SYSTEM


    general_log¶

    ▼ general_logとは¶

    一般ログを出力するか否かを設定する。

    [mysqld]
general_log = 1


    general_log_file¶

    ▼ general_log_fileとは¶

    一般ログの出力先のファイルを設定する。

    [mysqld]
general_log_file = mysql-general.log


    log_queries_not_using_indexes¶

    ▼ log_queries_not_using_indexesとは¶

    DBインデックスを使用するか否かを設定する。

    [mysqld]
log_queries_not_using_indexes = 0


    long_query_time¶

    ▼ long_query_timeとは¶

    スロークエリログを出力する場合に、スロークエリと見なす閾値秒数を設定する。

    [mysqld]
long_query_time = 3


    pid-file¶

    ▼ pid-fileとは¶

    プロセスIDが記載されたファイルの作成先を設定する。

    [mysqld]
pid-file = /var/run/mysqld/mysqld.pid


    secure-file-priv¶

    ▼ secure-file-privとは¶

    [mysqld]
secure-file-priv = /var/lib/mysql-files


    slow_query_log¶

    ▼ slow_query_logとは¶

    スロークエリログを出力するか否かを設定する。

    [mysqld]
slow_query_log = 1


    slow_query_log_file¶

    ▼ slow_query_log_fileとは¶

    スロークエリログの出力先のファイルを設定する。

    [mysqld]
slow_query_log_file = mysql-slow.log


    socket¶

    ▼ socketとは¶

    [mysqld]
socket = /var/lib/mysql/mysql.sock


    user¶

    ▼ userとは¶

    プロセスの実行ユーザー名を設定する。

    [mysqld]
user = mysql


    03-02. clientセクション¶

    clientセクションとは¶

    mysqlクライアントツールのclientプロセスのプールを設定する。


    default-character-set¶

    ▼ default-character-setとは¶

    デフォルトの文字コードを設定する。

    [client]
default-character-set = utf8mb4
